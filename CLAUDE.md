# CLAUDE.md for AI Coding Agents

## Project

l402: L402 client SDK for AI agent frameworks. Rust workspace with TypeScript and Python packages.

## Quick Start

```bash
cargo build          # Build all Rust crates
cargo test           # Run all Rust tests
cargo fmt --check    # Check formatting
cargo clippy         # Lint
cargo doc --no-deps  # Build docs

# TypeScript (Vercel AI SDK)
cd packages/l402-ai-sdk && yarn install && yarn test

# Python (LangChain)
cd packages/l402-langchain && pip install -e ".[dev]" && pytest
```

## Architecture

Hexagonal / Clean Architecture. Read `AGENTS.md` for the full breakdown.

### Rust Crates

- `l402-proto`: Protocol types (L402Challenge, L402Token, L402Error). No internal deps.
- `l402-core`: Client engine (L402Client), adapters (InMemoryTokenStore, BudgetTracker, Receipt). Depends on `l402-proto` for port traits.
- `l402-lnd`: LND gRPC adapter (implements LnBackend).
- `l402-cln`: CLN (Core Lightning) gRPC adapter (implements LnBackend).
- `l402-nwc`: Nostr Wallet Connect (NIP-47) adapter (implements LnBackend).
- `l402-swissknife`: SwissKnife REST adapter (implements LnBackend).
- `l402-mock`: Mock L402 server for testing.
- `l402-sqlite`: SQLite persistent token store (implements TokenStore).
- `l402-ffi`: C-compatible FFI layer for cross-language bindings.
- `l402-python`: Python bindings via PyO3/maturin.
- `l402-wasm`: WebAssembly bindings via wasm-pack.

### Bindings

- `bindings/l402-go`: Go bindings via CGo + l402-ffi.

### Packages (non-Rust)

- `packages/l402-ai-sdk`: Vercel AI SDK tools (TypeScript, yarn).
- `packages/l402-langchain`: LangChain Python tools (L402FetchTool, L402BudgetTool, PaymentCallbackHandler).

## Rules

1. **No TODOs or placeholders.** Implement fully or don't implement.
2. **Every public item must have documentation.** `#![warn(missing_docs)]` is enforced.
3. **Tests for every module.** Unit tests inline, integration tests in `tests/`.
4. **Conventional commits:** `feat:`, `fix:`, `chore:`, `test:`, `docs:`, `ci:`, `refactor:`
5. **One PR per logical change.** Squash merge. Clean history.
6. **No AI watermarks.** No `Co-authored-by: Claude`, no `Generated by AI`, nothing.
7. **Git config:** `Dario Anongba Varela <dario.anongba@gmail.com>`
8. **Error handling:** `thiserror` for typed errors. No `unwrap()` in library code. No `anyhow` in library crates.
9. **Async:** All port traits use `async_trait` (`?Send` on WASM). Core has no tokio dependency — uses `std::sync::RwLock` and `web_time::Instant`. Only `l402-lnd[grpc]` requires tokio.
10. **Format:** Run `cargo fmt` before committing. Run `cargo clippy` and fix all warnings.
11. **Use yarn** for TypeScript/Node.js packages (not npm).
12. **Never merge PRs autonomously in cron jobs.** Open PRs, leave for review.

## Dependencies (workspace-level)

Check `Cargo.toml` at workspace root for pinned versions. Don't add new dependencies without justification.

## Style Guide

- Prefer `impl` blocks over free functions
- Use `Self` in impl blocks where possible
- Group imports: std, external crates, internal crates
- Error variants should be descriptive (include context in the message)
- Prefer `&str` over `String` in function parameters where ownership isn't needed

## What NOT to Do

- Don't add `println!` for debugging, use `tracing`
- Don't use `unsafe` without extremely good reason
- Don't add feature flags unless architecturally necessary
- Don't break the public API without discussion
- Don't commit generated files (target/, *.lock for libraries)
- Don't build standalone binaries (MCP server, CLI) — L402sdk is a library/SDK
